## Maintenance Notes

This catalog is automatically maintained as new components are added to the library. When adding a new component:

1. Add component to appropriate category section
2. Include brief description of purpose/functionality
3. Link to primary code file (component.rs)
4. Update test status when tests are implemented
5. Mark theme support status (✅ or ❌)
6. Update component count statistics

For theme support integration, components should:
- Use `use_theme_context()` hook
- Subscribe to theme updates with reactive signals
- Support dynamic CSS variable injection
- Integrate with ThemeProvider context
